## What the platform does (v1 scope)

Per client business: import their customer list (CSV or booking-software export), run the 3-message sequence (personalized check-in, review request with the business's Google review link, click-checked reminder), track review-link clicks through our redirect domain, run a one-time reactivation drip over past customers (2-3 sends per 20 min, business hours), and email the owner a monthly before/after report. Review gating is never used: same link to everyone.

Out of v1 (phase 3+): AI auto-responses to reviews (needs Google Business Profile OAuth; the old OAuth client died in June per credentials notes), self-serve signup, Stripe billing, client dashboard.

## Architecture (existing stack, minimal new accounts)

- **Runtime:** Cloudflare Workers + D1 (multi-tenant schema: businesses, contacts, sequence_state, sends, clicks, reports) + cron triggers for the scheduler. Credentials on file.
- **Email:** Resend (key on file; the dead photobooths key is noted, use the live one).
- **SMS:** voip.ms (account + working token already on file, decision 2026-07-11) behind A2P 10DLC registration. voip.ms runs a white-glove registration: their messaging team files brand + campaign with the Campaign Registry ($4.50 brand + $15 campaign one-time, $1.50-30/mo by use case, carrier pass-through fees currently absorbed by voip.ms, up to 43 DIDs per campaign, approval up to ~15 days). Registration is still THE critical path; it is a carrier requirement and exists on any provider. voip.ms API is barebones (no built-in opt-out/delivery plumbing) so the engine owns STOP handling and suppression, which was already in scope. Fallback: Twilio if the pilot shows carrier filtering problems. Source: wiki.voip.ms 10DLC article.
- **Review links/QR:** Google Place ID lookup via Places API (google-api-keys.env), QR generated at onboarding.
- **Click tracking:** short redirect route on the worker domain per contact.
- **AI:** Anthropic API (key on file) for message personalization per business/industry and report narratives.
- **Compliance baked in:** opt-out handling (STOP for SMS, unsubscribe link for email), send-window limits, suppression list per business, only message customers the business actually served.
